MEERAHR SCHOOL ERP - PASSWORD RESET SOP (OPERATOR SCRIPT)

Purpose:
Securely reset user passwords with proper verification and audit trail.

Who can do this:
- Super Admin
- Admin

=================================================================
1) BEFORE RESET (MANDATORY VERIFICATION)
=================================================================
[ ] User identity verified (name + role + email/mobile)
[ ] Reason for reset captured
[ ] Operator name and time noted

If student request:
[ ] Roll number verified
[ ] Guardian phone verified

=================================================================
2) RESET USING PASSWORD RESET UTILITY
=================================================================
Step 1: Login as Admin/Super Admin
Step 2: Open Profile menu -> Password Reset Utility
Step 3: Select target user from dropdown (recommended)
Step 4: Choose one of below:
  - Verify only: click "Verify Password"
  - Reset password: click "Reset & Show Password"
Step 5: If needed, enter custom new password; else leave blank for system default
Step 6: Copy generated password and share securely
Step 7: Confirm user can login and is prompted to change password

=================================================================
3) SECURE SHARING RULES
=================================================================
[ ] Password never share on public group
[ ] Prefer direct call or secure private message
[ ] Ask user to change password immediately after first login
[ ] Do not store plain password in excel/chat permanently

=================================================================
4) AFTER RESET (AUDIT)
=================================================================
[ ] Reset completed confirmation captured
[ ] User name + role + timestamp recorded
[ ] Any issue logged in defect tracker

Operator Log Entry:
User: ____________________
Role: ____________________
Reset Time: ____________________
Operator: ____________________
Reason: ____________________

=================================================================
5) COMMON ISSUES & FIXES
=================================================================
Issue: Multiple users matched
Fix: Ask exact email/employee id/roll no and retry

Issue: Access denied to utility
Fix: Ensure operator is Admin or Super Admin

Issue: User says password not working
Fix: Re-run reset and verify no extra spaces while sharing

=================================================================
END OF SOP
